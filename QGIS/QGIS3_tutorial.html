名詞定義

  • 圖層(layer): 圖層是將空間資訊依主題繪製成的地圖，譬如便利商店的位置、捷運線路、森林公園...等各種不同的主題地圖，每一張主題地圖即代表一個圖層。
  • 圖徵(feature): 指圖層上的一筆資料，圖徵可以為點(便利商店)、線(捷運線)、面(森林公園)。
  • 圖徵屬性(attributes): 關於圖徵的各種變數，譬如一家便利商店的地址、名稱、員工人數、營業額...等。

    • 使用中研院批次定位處理(沒成功過)

    • 使用 google earth pro(需事先下載安裝)
      • https://www.google.com/earth/versions/#download-pro

google earth pro

  • 匯入資料

    1. [File] -> [Import] -> 選取欲輸入的檔案
    2. 選擇適當的 option -> [Next]
    3. 勾選 [This dataset does not contain latitude/longitude information, but street addresses] -> [Next]
    4. 勾選[This dataset contains one address field]，在 [Address field] 選擇 [hosp_addr] -> [Next]
    5. 建議所有資料類型都選擇 string -> [Finish]

  • 修正地址

    • 匯入資料後，會開始進行地址轉換。如果遇到無法轉換的地址，會跳出視窗 [google Earth could not geocode the following addresses:]
    • 可以修改 Address，或者使用 google earth 的建議，比較常見的問題是地址欄有樓層，或者有多個門牌號碼。
    • 所有地址都轉換後，會跳出 [Do you want to apply a style template to the features you ingested?] 勾選 [Yes]
    • [Set name file] 選擇 [醫事機構代碼] -> [OK] -> [cancel] 不儲存
    • 至此地址都已經匯入，可以在地球儀上看到醫院的位置。

  • 匯出資料

    • 在左側 [hosplist.csv] 底下的 [hosplist]，右鍵選取 [Save Place As...] -> 選擇 [kml] 格式存檔。

取得經緯度及重新投影

為何需要經緯度

  • 有些功能必須有地點的經緯度才能處理。譬如想使用 Openrouteservice 計算每家醫院的服務範圍時，就需要有經緯度。

為何需要重新投影

  • Google Earth 所使用的座標系為 EPSG:4326，但是 EPSG:4326 的距離單位是用經緯度定義，並不是很直覺。我們可以將其重新投影成以公尺為單位的座標系，譬如 EPSG:3857。

流程

  1. 右鍵點選 [hosplist] 圖層 -> [Export] -> [Save Features As...]
  2. [Format] 選擇 CSV 格式
  3. [File name] 選擇存檔位置，假設檔名為 hosp_loc
  4. [CRS] 選擇 EPSG:3857
  5. 不要勾選 [Add saved file to map]
  6. 選擇想保留的欄位
  7. [Layer Options] -> [GEOMETRY] -> 選擇 [AS_XY]
  8. 其他選項不用調整，選擇 [OK]，即存檔完成。
  9. 重新加入醫院位置：Delimited Text -> hosp_loc.csv，注意 [Geometry CRS] 必須選擇 EPSG:3857。

環域分析

什麼是環域

  • 環域分析是在點(商店)、線(輕軌線)、面(森林公園)等空間資料的外圍或內圍劃定區域範圍。譬如以便利商店為圓心，畫出半徑 500 公尺的圓為服務區域; 或者以輕軌線為軸心，畫出輕軌線兩旁 10 公尺內的區域為徵收範圍; 或者沿著森林公園的邊緣，向外畫出 1000 公尺的區域為保育區。

如何畫出環域

  1. [Vector] -> [Geoprocessing Tools] -> [Buffer]
  2. [Input layer]：選擇作為圓心的圖層
  3. [Distance]：選擇圓半徑及單位，注意，如果之前沒有改變投影方式，單位可能會被限定為經緯度。
  4. [Segments]：QGIS 會將此處輸入的數字乘以 4 再畫出對應的多邊型，譬如輸入 1，則會畫出一個正四邊形。
  5. [Buffered] 點選旁邊的 [...] -> [Save to File...] -> 選擇存檔位置。注意，如果沒有將圖層存檔，QGIS 關閉時該圖層也會消失。

路網服務區域分析

  1. [Plugins] -> [Manage and Install Plugins] -> 安裝 [OSM Tools]
  2. [開啟 OSM Tools] -> [Order Key!] 登入之後取得 API Key，把 API Key 貼回來。額度使用完畢後，要隔 24 小時才能再申請新 Key。
  3. [Isochrones] -> [distance] -> [Isochrone ranges] 輸入路徑長度 -> 勾選 [Use Layer] -> [Refresh List] -> 選擇作為起點的圖層
  4. 將新增的圖層存檔。注意，如果沒有將圖層存檔，QGIS 關閉時該圖層也會消失。：
    • 右鍵點選圖層 -> [Export] -> [Save Features As...]
    • [Format] 選擇 [ESRI Shapefile]
    • [File name] 選擇存檔位置
    • [CRS] 選擇 EPSG:3857
    • 勾選 [Add saved file to map]

計算區域內有多少點

  • 現在我們有兩個圖層，一個圖層標示出空間上的點，譬如醫院位置; 另一個圖層標示出空間上的區域，譬如每家醫院 3 公里區域。我們想計算每個區域內有多少點。

  1. [Vector] -> [Analysis Tools] -> [Count Points in Polygon...]。
  2. [Polygons] 選取標示區域的圖層。
  3. [Points] 選取標示點的圖層。
  4. [Count field name] 為計算結果的欄位名稱，可以自行更改。譬如 [hosp_3KM]
  5. [count] 點選旁邊的 [...] -> [Save to File...] -> 選擇存檔位置。
  6. 執行後會出現以區域為圖徵的新圖層，開啟屬性表格後，圖徵裡的 [hosp_3KM] 欄位即為計算結果。

Join

  • 假設有兩個圖層，第一個圖層有屬性 A、B，第二個圖層有屬性 C、D。我們發現，A 與 C 其實是相同的資料。我們可以利用 A、C 作為兩個圖層的 key，把 D 併入到第一個圖層裡。

  1. 選取第一個圖層 -> 右鍵選取 [Properties...]
  2. 選取 [Joins] -> 按下方的 + 號
  3. [Join layer] 選取第二個圖層
  4. [Join field] 選取第二個圖層裡的 C
  5. [Target field] 選取第一個圖層裡的 A
  6. [Joined Fields] 選取第二個圖層裡的 D
  7. 執行之後，就可以看到第一個圖層裡已經併入了 D 資料。

常用的空間分析功能(Geoprocessing Tools)

  • Buffer: 還域分析
  • Clip: 裁切，把圖層的一部分切出來
  • Dissolve: 融合，把圖層上數個多邊型融合成一個
  • Intersect: 取兩個圖層的交集
  • Union: 取兩個圖層的聯集
  • Difference: 取 A 圖層減去與 B 圖層交集的部份
