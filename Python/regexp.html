正規表達式

什麼是 regular expression?

我們在處理文本資料時，常會遇到需要判斷或找出資料裡是否有符合特定規則的字串。如何讓電腦能正確地找出字串，取決於我們能否精確地表達該字串的「規則」。正規表達式，就是用來表達字串規則的語法。

一個字元接一個字元地比對

"尋找/取代" 就是正規表達式最常見的應用。請試著開啟一個網頁，找出該網頁是否有 abc 這個字串。

不過，在學習正規表達式時，請不要將 abc 看成一個單位，而應該將其理解為 a，然後接著 b，然後接著 c，是三個有次序的字元。

比對時，電腦並不是去找 abc 這個字，而是先找到 a，再看是否接著 b，再看是否接著 c。如果都是，表示找到所要的字串。

雖然直接找 abc 跟依序找 a、b、c得到的結果都一樣，但是在下一節的例子裡，用「一個接一個比對」的方式，比較容易理解其意義。

開頭^與結尾$

本節介紹兩個特別的字元：^表示找出文本的開頭，$表示找出文本的結尾。

我們該怎麼理解 ^a 與 a$ 呢？

根據「一個接一個比對」的解讀，^a表示先找到文本的開頭，然後看是否接著a；而a$的意思表示先找到a，再看是否接著文本的結尾。

因此，如果對 abcabcabc 搜尋^abc會得到第一個abc；搜尋abc$會得到第三個abc；搜尋^abc$則找不到任何東西。

字元的集合[...]、[^...]

中括號夾著一串字元時[...]，只被視為一個字元，稱為 character class，在比對時，只要找到中括號裡的任一個字元即可。

舉例來說，[aA]bc表示先找到a或A都可以，再看是否接著 b，再看是否接著c。

[^...] 則表示只要「不是」中括號裡的任一個字元即可，舉例來說，[^aA]表示找出不是a也不是A的字元。

注意，中括號裡的^跟上一節的開頭符號相同，但是被放在中括號時，有不同的意義。

字元範圍 -

由上節說明我們知道，[12345]表示找到 1 到 5 的任一個數字即可，像這樣連續範圍的字元集合，可以用 - 來簡化，寫成 [1-5]。

任意字元.

. 用來表示任意字元，譬如a.c表示先找到a，再看是否緊接著任一個字元，再看是否接著c。

或 |

任意兩個正規表達式，可以用|連接，表示找出兩者之一即可。譬如 abc|Abc表示找出abc或者Abc。

組合(...)

要將兩個正規表達式組合成一個新的正規表達式單元，可以用(...)將它們包起來。譬如 (ab|Ab)c，小括號表示先找到ab或Ab，再看是否接著c。

綜合以上，可知以下幾種正規表達式意思完全相同：

  • [aA]bc
  • abc|Abc
  • (ab|Ab)c
  • (a|A)bc

可有可無?

若在一個正規表達式單元之後加?，表示有沒有找到這個單元都可以。譬如jpge?表示找出jpg或jpge都可以。do(es)?表示找出do或does都可以。

至少連續接著一個以上+

若在一個正規表達式單元之後加+，表示這個單元一定要連續接著出現至少一次以上。譬如abc+，表示ab之後一定要馬上緊接著一個以上的c，像是abc、abccc都可以。

可以連續接著一個以上，沒有也沒關係*

若在一個正規表達式單元之後加*，表示這個單元可以緊接著出現好幾次，不過沒有也沒關係。譬如abc*，表示ab之後可以接著一串c，也可以沒有c。因此ab或abccc都算是符合規則的字串。

最少 n 個，最多 m 個{n,m}

若在一個正規表達式單元之後加{n,m}，表示這個單元可以緊接著出現 n 到 m 次。譬如 abc{1,3}，表示ab之後可以緊接著 1 到 3 個c。

剛剛好有 n 個{n}

若在一個正規表達式單元之後加{n}，表示這個單元緊接著出現剛好 n 次。譬如 abc{3}，表示ab之後只能接剛好 3 個c，所以abccc符合規則，但abcccccc則不符合。

至少有 n 個{n,}

若在一個正規表達式單元之後加{n,}，表示這個單元緊接著出現至少 n 次。譬如 abc{3}，表示ab之後最少要有 3 個c。

從字串的開頭或結尾開始比對?=, ?<=

(?=abc)表示找出abc字串開頭的位置。這只會標註abc的開頭位置，而不是真的找出abc。(?=abc)abc表示先找出abc開頭的位置，再看是否依序接著a、b、c。因此，如果對 abcdef 搜尋(?=abc)abc，會得到 abc；搜尋(?=abc)def則找不到東西。

(?<=abc)則是找出abc字串結尾的位置。(?<=abc)abc表示先找出abc結尾的位置，再看是否依序接著a、b、c。因此，如果對 abcdef 搜尋(?<=abc)abc，會找不到東西；搜尋(?<=abc)def，則會得到 def。
