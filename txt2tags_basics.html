1. 開始之前

  • 使用任何文字編輯軟體編輯文件, 存檔時, 副檔名必須為 .t2t。
  • 一份 t2t 文件可以分成三大區塊： Header, Config, Body。Header 固定使用文件的前三行，緊接著是 Config，然後是 Body。

2. Header

  • 第 1 行固定為文件標題
  • 第 2 行為斜體字, 可放作者或 email。
  • 第 3 行為一般字體, 可放修改日期或版本資訊。
  • 如果要自動生成文件的修改日期，請加上 %%date(%Y.%m.%d)

3. Configuration

  • 可以指定 t2t 如何轉換成其他格式。指令固定為 %!keyword: value
  • %!target: html: 指定將文件轉換成 .html
  • %!options: --toc --toc-level 3: 指定轉換時加上目錄，並指定目錄層級。
  • %!encoding: UTF-8: 指定轉換時的編碼方式。
  • %!style: fancy.css: 指定轉換後的風格。
  • 更多說明請見 Settings, Configuration

5. 將 t2t 文件轉成 html 並放上網路

5.1. 編輯 t2t 文件

  • 使用任何文字編輯軟體編輯文件, 存檔時, 副檔名必須為 t2t。
  • 假設我們將檔案存在 MyFolder 資料夾，檔名為 FileName.t2t。

5.2. 將 t2t 輸出成 html

  • 開啟 Terminal。
  • 如果不在 FileName.t2t 所在的資料夾，則切換過去。
  • 執行 txt2tags -t html FileName.t2t
  • 若正常執行, 則 MyFolder 資料夾會出現 FileName.html 檔案。
  • 如果 t2t 文件裡已經設定好轉換時的格式，則執行 txt2tags FileName.t2t 即可。

5.3. 將 html 放在網路上

  • 先申請 Github pages。可參考此網頁說明。
  • 假設我們申請的 Github 帳號為 GitAccount, Github repository 名稱為 GitRepo。
  • 將 FileName.html 放到本機端的 GitRepo 資料夾。
  • 開啟 GitHub Desktop。
  • 輸入 commit 後, 執行 Commit to gh-pages。
  • 執行 sync。
  • 用瀏覽器開啟 https://GitAccount.github.io/GitRepo/FileName.html, 應該可以看到我們編輯好的網頁。
