install and upgrade

pip3 install -U otree

基本概念

  • otree 是基於 Python 與 Django 的架構，以實現在瀏覽器上，透過網頁做實驗的功能。關於 Django 的介紹，可參考 Django Girl 學習指南。
  • otree 可讓受試者在網頁上輸入資料，資料經過運算處理後，將結果呈現在網頁給受試者瀏覽。因此 otree 的實驗程式可以分成三個主要部分：
    1. 設定資料表格的欄位。
    2. 設計網頁呈現的樣式。
    3. 設定處理資料的流程。

範例0: 建立新的實驗

  • 範例 0 示範如何從無到有建立一個新的實驗，並讓受試者在本機端進行實驗。我們將專注在建立實驗到進行實驗的流程，而不會討論太多實驗程式的細節。我們將設計一個簡單問卷，讓受試者輸入姓名與年齡後，在網頁上顯示他們輸入的資料。

物件導向

  • 在 otree 裡，一場實驗稱為一個 Session。一場實驗有許多階段，一個階段稱為一個 Subsession。每個階段有許多步驟，每個步驟用一個頁面呈現，稱為 Page。

  • 來參加實驗的受試者稱為 Participant，受試者在每一個階段的實驗裡，可能跟其他受試者編為一組，稱為 Group，而受試者在實驗中扮演的角色，稱為 Player。

  • 在 Python 裡，所有東西都是物件(object)，類別(class)為物件的模板，物件是類別的一個例子。凡是物件都有屬性(attribute)跟方法(method)，屬性類似變數(variable)，而方法類似函數(function)。從資料結構來看，可以將類別理解為一個表格，物件即表格中的一列資料，屬性即資料的欄位，方法則是對資料的運算。

  • otree 裡的 Session、Subsession、Page、Group、Player、Participant 都是物件，都可以設定各自的屬性與方法。而在 otree 裡，這些物件有上下層關係。它們的關係如下圖：

  • 給定某個物件，我們稱此物件為自己(self)。上層物件對自己而言，是自己的屬性；而自己可以透過方法取得下層物件。譬如給定某個 Group 為 self，self.subsession 這個屬性用來表示此 Group 所在的 Subsession，而 self.get_players() 這個方法可以取得此 Group 裡的 Player。更詳盡的範例如下：

  • 範例：models.py 

    class Subsession(BaseSubsession):
    def example(self):

        # current subsession object
        self

        # parent objects
        self.session

        # child objects
        self.get_groups()
        self.get_players()

        # accessing previous Subsession objects
        self.in_previous_rounds()
        self.in_all_rounds()

class Group(BaseGroup):
    def example(self):

        # current group object
        self

        # parent objects
        self.session
        self.subsession

        # child objects
        self.get_players()

class Player(BasePlayer):
    def example(self):

        # current player object
        self

        # method you defined on the current object
        self.my_custom_method()

        # parent objects
        self.session
        self.subsession
        self.group
        self.participant

        self.session.config

        # accessing previous player objects
        self.in_previous_rounds()

        # equivalent to self.in_previous_rounds() + [self]
        self.in_all_rounds()

  • 範例：pages.py

    class MyPage(Page):
    def example(self):

        # current page object
        self

        # parent objects
        self.session
        self.subsession
        self.group
        self.player
        self.participant
        self.session.config

重啟資料庫與伺服器

先進入 your_project_name 資料夾, 輸入

$otree reserdb 
$otree runserver

使用 Heroku

$git add . 
$git commit -am 'my comment' 
$git push heroku master 
$heroku run otree resetsb ## 重設資料庫 
$heroku open
