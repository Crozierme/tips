範例：最後通牒實驗

  • 最後通牒實驗每組有兩位受試者，分別扮演提案者與回應者。每一組每一回合提案者有 20 元稟賦，提案者先提案分多少錢給回應者，回應者可以選擇接受或拒絕。若回應者接受提案，則按提案分配。若回應者拒絕提案，兩人都得到 0 元。如此進行 3 回合後，實驗結束。

  1. 建立 app
  2. 修改 model.py
  3. 修改 template
  4. 修改 page.py
  5. 修改 settings.py

1. 建立 app

otree startapp my_ultimatum
  • 在my_project底下輸入上述指令，otree 就會在my_project內新增my_ultimatum資料夾。

2. 修改 model.py

  • model.py的功能為定義資料表格的欄位與類型，也可以用來定義實驗的經濟模型。
2.1 class Constants(BaseConstants)

  • 完整的Constant class 設定如下

    class Constants(BaseConstants):
    name_in_url = 'my_ultimatum'
    players_per_group = 2
    num_rounds = 3
    endowment = c(20)
2.2 class Player(BasePlayer)

  • Player class 設定與 player 有關的資料欄位與類型。在最後通牒遊戲裡，提案者提案分多少錢給回應者，回應者選擇接受或拒絕。因此直觀上我們認為Player class 應該有兩個欄位：

    class Player(BasePlayer):
    offer_to_responser = models.CurrencyField(min=0, max=Constants.endowment)
    offer_accept = models.BooleanField()

    這個設定的問題在於，我們會讓所有受試者的資料都有兩個欄位。但其實應該只有提案者有offer_to_responser欄位，只有回應者有offer_accept欄位。因此我們應該將提案者與回應者視為一組，而每一組的資料都有這兩個欄位。

2.3 class Group(BaseGroup)

  • class Group(BaseGroup)記錄與 group 有關的資料欄位與類型。如前所言，每組需要記錄offer_to_responser與offer_accept兩個欄位。

    class Group(BaseGroup):
    offer_to_responser = models.CurrencyField(min=0, max=Constants.endowment)
    offer_accept = models.IntegerField(
        choices=[
            [0, '拒絕'],
            [1, '接受'],
        ]
    )

    此處我們設定offer_accept的類型為整數，並讓每個數值都有相對應的文字標籤。如果沒有設定文字標籤，則回應者看到的選單會以數字呈現，因此設定文字標籤可以方便回應者選擇。另外，這個文字標籤只會在選單出現，其他網頁呼叫這個變數時，還是會呈現數字。在 otree 裡，當我們幫某變數foo設定好choices後，otree 會自動新增get_foo_display()函數讓我們取得該變數的文字標籤。因此如果要讓offer_accept以文字標籤方式呈現，可以使用函數get_offer_accept_display()。詳情請見底下page.py的設定。註：

4. 修改 page.py

  • 我們在template有四個頁面，但是Instructions.html並不是獨立的頁面，因此在page.py裡只需要設定另外三個相對應的 class 來處理這三個頁面。除此之外，實驗需要另外兩個WaitPage，一個是回應者等待提案者提案，一個是回應者做出選擇後，等待程式計算結果。

  • Proposer.html包含一個表單讓提案者提案，需要在class Proposer(Page)設定表單所對應的欄位。此外Proposer.html只有提案者會看到，可以使用Pageclass 內建的函數名稱is_displayed()，我們需要定義該函數內容，otree 會在該函數的return值為 True 時顯示本頁。假設我們讓組內 id 為 1 的受試者為提案者，則我們可以定義is_displayed()的return值為self.player.id_in_group == 1，這樣當受試者的組內 id 為 1 時，return值即為 True。

    class Proposer(Page):

    form_model = 'group'
    form_fields = ['offer_to_responser']

    def is_displayed(self):
        return self.player.id_in_group == 1

  • Responser.html包含一個表單讓回應者選擇拒絕或接受，需要在class Responser(Page)設定表單所對應的欄位。同樣地，Responser.html只顯示給回應者看，所以需要用is_displayed()設定顯示條件。

    class Responser(Page):

    form_model = 'group'
    form_fields = ['offer_accept']

    def is_displayed(self):
        return self.player.id_in_group == 2

  • 如前所言，我們需要設定兩個WaitPage，其中一個是回應者等待提案者提案，我們將其稱為class WaitForP1(WaitPage)，此時不需要任何計算。另一個是程式等回應者做出選擇後，計算結果並傳送到Results.html呈現，我們將其稱為class ResultsWaitPage(WaitPage)。我們使用WaitPageclass 內建的函數名稱after_all_players_arrive()定義回應者做出選擇後的計算，它會在所有人都進入等待頁面後自動執行。

    class ResultsWaitPage(WaitPage):

   def after_all_players_arrive(self):
        group = self.group
        p1 = group.get_player_by_id(1)
        p2 = group.get_player_by_id(2)
        p1.payoff = group.offer_accept * (Constants.endowment - group.offer_to_responser)
        p2.payoff = group.offer_accept * group.offer_to_responser

  • 我們在前面的Result.html設定過responser_choice，這個變數不存在model.py裡。如果要傳送不存在的變數值傳送到Result.html呈現，則可以在class Results(Page)使用Pageclass 內建的函數名稱def vars_for_template()，該函數的return值必須為 {target_variale : target_value} 的 dict，otree 會將return值呈現在該頁面。在本例中，target_variabl即頁面上的responser_choice。而target_value是將group.offer_accept的值從數值轉成我們在model.py設定好的文字標籤，可以用函數get_offer_accept_display()來取得文字標籤：

    class Results(Page):
    def vars_for_template(self):
        return {
            'responser_choice':self.group.get_offer_accept_display()
        }

  • 最後，我們設定頁面呈現的順序如下：

        page_sequence = [
        Proposer,
        WaitForP1,
        Responser,
        ResultsWaitPage,
        Results,
    ]

  • 完整的page.py如下：

    from otree.api import Currency as c, currency_range
from ._builtin import Page, WaitPage
from .models import Constants

class Proposer(Page):

    form_model = 'group'
    form_fields = ['offer_to_responser']

    def is_displayed(self):
        return self.player.id_in_group == 1

class Responser(Page):

    form_model = 'group'
    form_fields = ['offer_accept']

    def is_displayed(self):
        return self.player.id_in_group == 2

class WaitForP1(WaitPage):
    pass

class ResultsWaitPage(WaitPage):

    def after_all_players_arrive(self):
        group = self.group
        p1 = group.get_player_by_id(1)
        p2 = group.get_player_by_id(2)
        p1.payoff = group.offer_accpet * (Constants.endowment - group.offer_to_responser)
        p2.payoff = group.offer_accpet * group.offer_to_responser

class Results(Page):
    def vars_for_template(self):
        return {
            'responser_choice':self.group.get_offer_accept_display()
        }

page_sequence = [
    Proposer,
    WaitForP1,
    Responser,
    ResultsWaitPage,
    Results,
]
