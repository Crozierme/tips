範例0: 建立新的實驗

範例 0 示範如何從無到有建立一個新的實驗，並讓受試者在本機端進行實驗。我們將設計一個簡單問卷，讓受試者輸入姓名與年齡後，在網頁上顯示他們輸入的資料。流程如下：

  1. 建立 project
  2. 建立 app
  3. 修改 model.py
  4. 修改 template
  5. 修改 page.py
  6. 修改 settings.py
  7. 啟動 server

1. 建立 project

$otree startproject my_project
  • 輸入上述指令後，otree 會在目前目錄底下建立一個my_project的資料夾。project 並不等於一個實驗，而是裝許多實驗程式的資料夾。個別的實驗則被稱為 app。
  • startproject時，可以選擇是否要安裝 otree 內建的實驗範例。

2. 建立 app

$cd my_project
$otree startapp my_app
  • 輸入上述指令，otree 就會在my_project底下新增my_app資料夾。
  • my_app資料夾包括model.py、pages.py兩個檔案以及templates資料夾。
    • model.py用來設定資料表格的欄位與類型。
    • pages.py設定資料如何處理與運算。
    • templates資料夾裡放置網頁樣式，決定網頁的呈現方式。

3. 修改 model.py

  • 我們第一個實驗是讓受試者填寫姓名跟年齡，我們的資料應該包括一個受試者表格，且表格裡有姓名與年齡兩個欄位。

  • 我們可以在model.py的class Player(BasePlayer)設定這兩個欄位，其中name為字串格式，age與birthyear為整數格式。

    class Player(BasePlayer):
    name = models.StringField()
    age = models.IntegerField()
    birthyear = models.IntegerField()

  • 我們暫時不去修改model.py其他部分，因此完整的model.py如下：

    from otree.api import (
    models, widgets, BaseConstants, BaseSubsession, BaseGroup, BasePlayer,
    Currency as c, currency_range
)

author = 'Your name here'

doc = """
Your app description
"""

class Constants(BaseConstants):
    name_in_url = 'my_app'
    players_per_group = None
    num_rounds = 1

class Subsession(BaseSubsession):
    pass

class Group(BaseGroup):
    pass

class Player(BasePlayer):
    name = models.StringField()
    age = models.IntegerField()

4. 修改 template

  • 我們的實驗包含兩個頁面，第一個是讓受試者填答的問卷頁面，第二個是顯示結果的頁面。我們要設計這些頁面呈現的樣式，並且放在templates/my_app/裡。

  • 在templates/my_app/已經有兩個內建的頁面MyPage.html及Results.html，我們直接將其修改如下：

  • 問卷頁面MyPage.html：

    {% extends "global/Page.html" %}
{% load static otree %}

{% block title %}
    請輸入以下資訊：
{% endblock %}

{% block content %}

    {% formfield player.name label = "姓名"%}

    {% formfield player.age label = "年齡"%}

    {% next_button %}

{% endblock %}

  • 結果頁面Results.html：

    {% extends "global/Page.html" %}
{% load static otree %}

{% block title %}
    你輸入的資訊
{% endblock %}

{% block content %}

    <p>你的名字是{{ player.name }}</p> 
    <p>你的年齡是{{ player.age }}</p> 

    {% next_button %}
{% endblock %}

  • 以上兩個頁面裡的player.name與player.age，即對應到我們在model.py的class Player(BasePlayer)設定的兩個欄位。

5. 修改 page.py

  • 因為網頁本身並沒有計算與處理資料的功能，我們需要在page.py裡設定各個網頁怎麼處理資料。舉例來說，我們讓受試者在MyPage.html輸入姓名與年齡，當受試者按下發送後，MyPage.html並不會將剛剛輸入的資訊放到資料庫裡，我們需要在page.py裡，設定如何把資訊放到資料庫裡指定的欄位。
  • 因為我們有兩個頁面，所以在page.py裡需要兩個相對應的 class 來處理這兩個頁面，class 的名稱必須與網頁名稱相同。

  • 我們在class MyPage(Page)裡，設定受試者在MyPage.html輸入的資訊對應到資料庫的哪些欄位。設定的方式如下：

    class MyPage(Page):
    form_model = 'player'
    form_fields = ['name', 'age']

    form_model對應model.py的class Player(BasePlayer)，form_fields對應欄位名稱。

  • Result.html並不需要處理資料，所以在class Results(Page)不需要做任何設定。

  • 內建的page.py還有其他頁面 class，目前用不到，可以先刪除。

  • 最後，我們在page_sequence設定頁面出現的順序為MyPage後接著Results。完整的page.py如下：

    from otree.api import Currency as c, currency_range
from ._builtin import Page, WaitPage
from .models import Constants

class MyPage(Page):
    form_model = 'player'
    form_fields = ['name', 'age']

class Results(Page):
    pass

page_sequence = [
    MyPage,
    Results
]

6. 修改 settings.py

  • 在啟動實驗之前，我們需要到my_project裡修改settings.py。在settings.py裡的SESSION_CONFIGS加入我們剛剛寫好的 app，設定如下：

    SESSION_CONFIGS = [
    {
        'name': 'my_app',
        'num_demo_participants': 3,
        'app_sequence': ['my_app'],
    },
]

    7. 啟動 server

    $otree devserver

  • 輸入上述指令後，用瀏覽器開啟 http://localhost:8000/，就可以看到我們新建立的實驗了。
