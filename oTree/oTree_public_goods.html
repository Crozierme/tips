範例：公共財實驗

  • 公共財實驗每組有四位受試者，每個人每回合一開始都有 20 元稟賦，可以自由捐獻。將同組所有受試者的捐獻加總後，乘上兩倍，再平分給四位受試者，此即公共財帶給受試者的效益，加上未捐出的稟賦，即為受試者在這一回合的報酬。如此進行 3 回合後，實驗結束。

1. 建立 app

otree startapp my_public_goods
  • 在my_project底下輸入上述指令，otree 就會在my_project內新增my_public_goods資料夾。

2. 修改 model.py

  • model.py的功能為定義資料表格的欄位與類型，也可以用來定義實驗的經濟模型。
2.1 class Constants(BaseConstants)

  • Constant class 用來放固定不變的參數。其中已經內建三個必要屬性：

    • name_in_url：顯示在實驗連結的名稱
    • players_per_group：每組人數
    • num_rounds：實驗一共進行幾回合

  • 在公共財實驗裡，每一期每一人的稟賦固定不變，公共財的加乘倍數也是固定為 2，這兩個固定不變的常數，適合放在Constant class 裡。設定的方式如下：

    endowment = c(20)
multiplier = 2

  • 注意，endowment不是設定為endowment = 20，而是endowment = c(20)。c(n)表示設定該變數為 otree 特有的類型：CurrencyField，且值為n。與一般數值類型的差別在於，當CurrencyField類型的數值顯示給受試者時，會自動加上貨幣符號，譬如$20或20點。可以在settings.py的REAL_WORLD_CURRENCY_CODE跟LANGUAGE_CODE settings更改貨幣符號。

  • 你也可以在model.py裡將c()改成其他名稱，在model.py的開頭有一段程式為

    from otree.api import (
    models, widgets, BaseConstants, BaseSubsession, BaseGroup, BasePlayer,
    Currency as c, currency_range
)

    c()即來自import Currency as c的c。

  • 完整的Constant class 設定如下

    class Constants(BaseConstants):
    name_in_url = 'my_public_goods'
    players_per_group = 4
    num_rounds = 3

    endowment = c(20)
    multiplier = 2
2.2 class Player(BasePlayer)

  • Player class 設定與 player 有關的資料欄位與類型。公共財實驗需要記錄每人的捐獻contribution。otree 有一個特殊的變數類型CurrencyField，用來處理與實驗法幣有關的變數。因此我們將contribution的類型設定為CurrencyField，且限制它的範圍在 0 到endowment之間。 

    class Player(BasePlayer):
    contribution = models.CurrencyField(min=0, max=Constants.endowment)

  • 每個人的報酬payoff也是與 player 有關的資料欄位，但是這已經內建在 BasePlayer，所以不需要再另外設定。

2.3 class Group(BaseGroup)

  • class Group(BaseGroup)記錄與 group 有關的資料欄位與類型。公共財實驗需要記錄每一組加總的公共財數量total_contribtunio，以及該組每人平均分得多少公共財效益individual_share，因為這兩個變數都是處理實驗法幣，所以同樣將類型設定為CurrencyField。

    class Group(BaseGroup):
    total_contribution = models.CurrencyField()
    individual_share = models.CurrencyField()
2.3 完整的 model.py
  • from otree.api import (
    models, widgets, BaseConstants, BaseSubsession, BaseGroup, BasePlayer,
    Currency as c, currency_range
)

author = 'Your name here'

doc = """
Your app description
"""

class Constants(BaseConstants):
    name_in_url = 'my_public_goods'
    players_per_group = 4
    num_rounds = 3

    endowment = c(20)
    multiplier = 2

class Subsession(BaseSubsession):
    pass

class Group(BaseGroup):
    total_contribution = models.CurrencyField()
    individual_share = models.CurrencyField()

class Player(BasePlayer):
    contribution = models.CurrencyField(min=0, max=Constants.endowment)

3. 修改 template

  • 這個實驗一共有兩個頁面，一個是受試者決定捐獻多少錢的頁面Contribute.html，一個是捐獻後的結果頁面Result.html。設計好這兩個頁面樣式後，將它們放在templates/my_public_goods/裡。

  • 捐獻頁面Contribute.html：

    {% extends "global/Page.html" %}
{% load static otree %}

{% block title %} 捐獻 {% endblock %}

{% block content %}

    <p>
        每組共有{{ Constants.players_per_group }}人，每人有{{ Constants.endowment }}稟賦，所有人的捐獻加總後將乘上{{ Constants.multiplier }}倍。
    </p>

    {% formfield player.contribution label="請輸入你的捐獻" %}

    {% next_button %}

{% endblock %}

  • 結果頁面Contribute.html：

    {% extends "global/Page.html" %}
{% load static otree %}

{% block title %} 公共財數量 {% endblock %}

{% block content %}

    <p>
        你有{{ Constants.endowment }}稟賦，你捐獻{{ player.contribution }}。你們這組的總捐獻為{{ group.total_contribution }}，因此每人可分得公共財效益{{ group.individual_share }}。
        所以你的報酬是{{ player.payoff }}.
    </p>

    {% next_button %}

{% endblock %}

4. 修改 page.py

  • 我們在template有兩個頁面，因此在page.py裡也需要兩個相對應的 class 來處理這兩個頁面。

  • Contribute.html包含一個表單讓受試者填資料，在class Contribute(Page)裡，設定表單資訊對應到資料庫的哪些欄位。設定的方式如下：

    class Contribute(Page):
    form_model = 'player'
    form_fields = ['contribution']

  • Result.html並不需要處理資料，所以在class Results(Page)不需要做任何設定。

  • 在實驗進行時，受試者在Contribute.html決定捐獻數字後，不能馬上看到結果，需要等其他人都完成捐獻後，算出公共財效益，然後才進入Results.html將結果呈現給受試者瀏覽。因此我們需要在Contribute.html跟Results.html之間插入一個頁面來處理等待與計算。在 otree 裡class ResultsWaitPage(WaitPage)即負責這個功能。

  • WaitPage class 已經內建了一個函數名稱after_all_players_arrive()，當所有人都進到等待頁面後，程式會自動執行函數內容。請注意，otree 只是內建了這個函數名稱，但沒有定義函數內容，需要自己定義。

    class ResultsWaitPage(WaitPage):

    def after_all_players_arrive(self):
        group = self.group
        players = group.get_players()
        contributions = [p.contribution for p in players]
        group.total_contribution = sum(contributions)
        group.individual_share = group.total_contribution * Constants.multiplier / Constants.players_per_group
        for p in players:
            p.payoff = Constants.endowment - p.contribution + group.individual_share

  • 最後，我們要設定頁面呈現的順序如下：

    page_sequence = [
    Contribute,
    ResultsWaitPage,
    Results
]

6. 倒數計時

  • otree 可以在頁面上增加倒數計時，當時限為 0 時，會自動完成指定動作並跳到下一頁。假設我們要在捐獻時加入倒數計時 15 秒，時間到如果受測者還沒有決定捐獻，則由電腦隨機決定。則可以在Page.py裡的class Contribute(Page)做以下設定：

  • 設定倒數時限：在class Contribute(Page)加入以下指令

    timeout_seconds = 15

  • 設定時限為 0 時的動作：在Page.py加上import random，並在class Contribute(Page)裡定義函數before_next_page()如下

    def before_next_page(self):
    if self.timeout_happened == True:
        self.player.contribution = random.randint(0, 20)

    before_next_page()是 otree 內建的函數名稱，函數內容可以由我們定義，它將會在受試者離開本頁且進入下一頁之前自動執行。timeout_happend是 otree 內建的屬性，當頁面倒數時限為 0 時，timeout_happend會自動設定為True。

  • 注意，一旦設定了timeout_seconds，即使沒有設定before_next_page()，程式還是會自動跳到下一頁。

  • 如果我們只想給受試者提醒，但不要跳至下一頁的話，則不要在class Contribute(Page)裡加上timeout_seconds，而改在Contribute.html加入以下段落：

    {% block scripts %}

    <script>
      setTimeout(
    function () {
        alert("時間到，請做決定");
    },
    15*1000 // 15 seconds
);
    </script>

{% endblock %}

    注意：這時候頁面不會出現倒數計時器，當受試者在頁面超過15秒，會跳出視窗提醒受試者做決定。

7. 分組

  • 如果要在實驗中將受試者隨機分組，首先在settings.py的SESSION_CONFIGS修改受試者的人數：

    SESSION_CONFIGS = [
    {
        'name': 'my_public_goods',
        'num_demo_participants': 8,
        'app_sequence': ['my_public_goods'],
    },
    ]

  • 接著在model.py的class Subsession(BaseSubsession)加入以下設定：

    class Subsession(BaseSubsession):
    def creating_session(self):
        self.group_randomly()

    creating_session()是Subsessionclass 內建的函數名稱，函數內容可以由我們定義，它會在每次開啟Subsession時自動執行。group_randomly()則是內建的函數，它會將受試者隨機分組。

  • 註：在 otree 裡，將一個實驗 app，按指定的頁面順序，從頭到尾執行一次，稱為一個Subsession。譬如我們在model.py裡設定num_rounds = 3，公共財實驗會重複玩 3 遍，這次實驗就有 3 個Subsession。
